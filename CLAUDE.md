## 2. 每日操作流程

**注意事项**

**每次分析/操作前必须重新运行命令获取最新数据，严禁使用会话中的缓存数据，股价随时变化，缓存数据可能已过期导致错误决策**
所有python脚本编码都是utf-8, 运行python脚本请使用`python -X utf8`
读取xlsx时，请使用`python -X xlsx_to_json.py <filename>`转成json格式方便你读取

**操作流程**

1. **读取当前数据**：读取 `stocks.xlsx` 中的最新股票技术指标
2. **读取历史数据**：读取 `data/` 目录下最近30天的**所有**历史数据文件，请不要遗漏
3. **趋势对比分析**：对比7天、30天前后技术指标变化，评估趋势持续性
4. **分析持仓**：查看 `MY.md` 当前持仓，结合历史数据趋势评估持仓质量
5. **回顾历史经验**：查看 `HISTORY.md` 查看历史操作记录和分析
6. **识别信号**：基于当前+历史数据，识别买入/卖出/持仓信号
7. **生成操作**：确定今日买入/卖出/持仓的股票和价格
8. **记录日志**：将今日操作写入 `HISTORY.md` 的底部

## 4. 大盘研判框架

**每次操作前必须先读 `index.xlsx`，大盘状态决定仓位上限和操作力度。**

### 4.1 大盘状态分级

| 状态 | 判断依据 | 仓位上限 | 操作基调 |
|------|---------|---------|---------|
| 强势 | 上证DIF>0 + MA多头 + 量比≥1 | **80%** | 积极持仓，可追强势股 |
| 震荡 | 上证DIF正负交织 / MA混乱 / 量比0.5-1 | **65%** | 守仓为主，精选个股 |
| 弱势 | 上证DIF<0 + MA空头 / 量比<0.5 | **50%** | 轻仓防守，止损优先 |
| 恐慌 | 上证单日跌幅>2% + DIF加速下行 | **30%** | 停止新买，止损清弱势仓 |

### 4.2 历史大盘趋势读取

`indices/` 目录保存每日大盘快照（格式：`index_YYYYMMDD_HHMMSS.xlsx`），分析时须对比近期趋势：

- **读取范围**：最近 7 天的 `indices/` 文件（每天取最新一个）
- **核心对比维度**：

| 维度 | 看什么 | 意义 |
|------|--------|------|
| DIF 方向 | 7 天内 DIF 持续上行/下行 | 趋势加速或衰竭 |
| DIF 零轴 | 从负区转正 / 从正区跌负 | 牛熊切换信号 |
| 价格 vs MA20 | 是否站上/跌破 MA20 | 中期趋势判断 |
| 量比趋势 | 量比是否持续萎缩/放大 | 资金参与意愿 |

- **结论输出**：用"7日前→今日"格式标注 DIF 变化，判断趋势是"加速上行/减速/横盘/加速下行"

### 4.3 大盘联动操作规则

1. **新建仓禁区**：上证 DIF < -30（深度负区）时，禁止新建任何仓位
2. **仓位压缩**：大盘弱势时，单票仓位上限从 30% 压缩至 20%
3. **轻仓原则**：大盘量比 < 0.5 时，个股即使技术面良好，建仓不超过标准仓的 50%
4. **板块选择**：优先选与大盘相关性低的板块；创业板 DIF > 0 + MA多头时，可聚焦科技/成长股
5. **止损加速**：大盘破位（上证跌破 MA20）时，个股止损线上移 2%（提前止损）
