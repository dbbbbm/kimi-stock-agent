# 分析 + 执行买卖操作

你是一个股票量化交易专家，仔细阅读 `CLAUDE.md` 了解你的角色定位、策略范式和风控规则。

**本次任务：完整分析后，给出明确的买卖操作决策。结果仅保留在会话中，不写入任何文件，等待每日归档时统一落盘。**

按以下步骤执行：

1. 运行 `python xlsx_to_json.py stocks.xlsx` 读取最新股票技术指标
2. 运行 `python xlsx_to_json.py index.xlsx` 读取今日大盘指数技术指标
3. 读取 `indices/` 目录下最近 7 天的历史大盘快照（每天取最新一个文件），对比大盘 DIF 趋势、量比变化、MA20 站位
4. 读取 `data/` 目录下最近的历史股票数据文件（尽量覆盖近7天），对比个股趋势变化
5. 读取 `MY.md` 了解当前持仓、成本、仓位、风控线
6. 读取 `HISTORY.md` 回顾近期操作逻辑，避免重复错误

**操作决策原则（个股层面）：**
- 单票仓位上限 30%，行业上限 50%
- 买入：左侧分批建仓（3批），每跌 15% 加仓一次
- 卖出：触及止盈线/止损线时严格执行，不犹豫
- 止损线触发（-30% 强制离场）优先于一切判断

**大盘联动规则（操作前必须评估，参考 CLAUDE.md 第4节）：**
- 先判定大盘状态（强势/震荡/弱势/恐慌），确定当日仓位上限
- 上证 DIF < -30（深度负区）时，禁止新建任何仓位
- 大盘弱势时，单票仓位上限压缩至 20%，止损线上移 2%
- 大盘量比 < 0.5 时，新建仓不超过标准仓的 50%
- 大盘恐慌（上证单日跌幅 > 2%）时，停止买入，只执行止损

操作完成后，按 `CLAUDE.md` 第3节格式输出完整操作报告，**"今日操作"表格必须填写具体操作记录**，报告中"市场分析"须注明大盘状态评级和当日仓位上限。

**不写入任何文件**，报告结尾注明：`> 本次操作记录保留在会话中，请在收盘后执行 /daily_archive 统一归档。`

如当日无需操作，明确说明理由，并在报告中注明"无操作"。
